![Docker](https://img.shields.io/badge/docker-ready-blue.svg)

**AI-SDLC** is an enterprise-grade, fully autonomous, model-agnostic platform that builds complex software end-to-end. From raw idea to production-ready, security-hardened, self-healing code—all executed by a swarm of 15 specialized AI agents.

## 🌟 Key Features

* **Fully Autonomous Execution:** Pass an idea and get back a complete software system. No human intervention required (unless you enable Human-In-The-Loop gates).
* **15 Specialized Agents:** Ideation, Business Analysis, Architecture, UX Design, Engineering, Data Engineering, QA, Security, DevOps, Self-Healing, Chaos Engineering, Compliance, FinOps, and Maintenance.
* **NoOps & Self-Healing:** Everything built is resilient by default. Includes circuit breakers, auto-remediation playbooks, health probes, and chaos engineering tests.
* **Security First:** Built-in SAST, secret scanning, dependency checking, and Zero-Trust Kubernetes policy generation (mTLS, RBAC, NetworkPolicies).
* **Model-Agnostic:** Works with OpenAI, Anthropic, Google Gemini, Ollama, or any OpenAI-compatible endpoint.
* **Multi-Layer Memory:** 4-layer cognitive memory (Working, Episodic, Semantic, Procedural) backed by ChromaDB. The system learns from every project.
* **MCP Server Integration:** Exposes the entire SDLC pipeline as Model Context Protocol (MCP) tools. Connect Cursor, Claude Desktop, or Continue and use AI-SDLC as a native tool.
* **Self-Hosted Stack:** Complete Docker Compose setup with API, MCP, Redis, Postgres, Nginx, Prometheus, and Grafana.

## 🚀 Quick Start

### 1. Install via pip

```bash
pip install -e .
```

### 2. Run the CLI

```bash
export OPENAI_API_KEY="sk-..."

# Build a complete project from a single prompt
aisdlc run "Build a multi-tenant SaaS CRM platform for real estate agents"

# Or use a different model
aisdlc run "Build a trading bot" --provider anthropic --model claude-3-5-sonnet-20240620
```

### 3. Self-Hosted Docker Stack

To run the full enterprise stack with API, UI, Memory, and Monitoring:

```bash
export OPENAI_API_KEY="sk-..."
export POSTGRES_PASSWORD="secure_password"

docker-compose up -d
```

Services available:
* REST API: `http://localhost:8000/docs`
* MCP Server: `http://localhost:8765/mcp`
* Grafana Dashboards: `http://localhost:3000`

## 🧠 The Agent Swarm

AI-SDLC uses a DAG-based orchestration engine (the **Conductor**) to manage the flow between agents:

1. **Ideation Agent:** Expands the raw idea into a product vision.
2. **BA Agent:** Writes the comprehensive Product Requirements Document (PRD).
3. **Architect Agent:** Designs the system architecture, component diagrams, and tech stack.
4. **UX Agent:** Designs the user experience, wireframe specs, and user flows.
5. **Data Engineer Agent:** Designs the database schema, data pipelines, and caching strategy.
6. **Security Agent:** Threat modeling, zero-trust policies, and continuous scanning.
7. **Engineer Agent:** Writes the actual application code across frontend and backend.
8. **QA Agent:** Generates unit, integration, and E2E test suites.
9. **DevOps Agent:** Generates Dockerfiles, Kubernetes manifests, and CI/CD pipelines.
10. **Self-Healing Agent:** Injects circuit breakers, health checks, and auto-remediation code.
11. **Chaos Agent:** Designs chaos engineering experiments to validate resilience.
12. **Compliance Agent:** Ensures GDPR/SOC2/HIPAA compliance in code and infrastructure.
13. **FinOps Agent:** Optimizes cloud resource usage and estimates costs.
14. **Maintenance Agent:** Generates runbooks, SLA definitions, and upgrade paths.

## 🔌 Model Context Protocol (MCP)

AI-SDLC runs an MCP server, allowing any MCP-compatible client to use the platform as a tool.

Start the server:
```bash
aisdlc mcp start --transport stdio
```

Add to your `claude_desktop_config.json` or Cursor settings:
```json
{
  "mcpServers": {
    "ai-sdlc": {
      "command": "aisdlc",
      "args": ["mcp", "start", "--transport", "stdio"],
      "env": {
        "OPENAI_API_KEY": "sk-..."
      }
    }
  }
}
```

## 🛡️ Security & NoOps

AI-SDLC doesn't just write code; it writes *production-ready* code.
* **Auto-Remediation:** If a pod runs out of memory, the generated NoOps engine automatically captures a heap dump and restarts it.
* **Circuit Breakers:** All external API calls generated by the Engineer Agent are wrapped in thread-safe circuit breakers.
* **Zero-Trust:** Every Kubernetes deployment gets a strict NetworkPolicy, read-only root filesystem, and Istio mTLS configuration.
