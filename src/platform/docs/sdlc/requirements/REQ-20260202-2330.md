## 1. Executive Summary

### 1.1 Problem Statement

The Deltek Catalyst Visual Designer currently supports only 5 basic AWS node types (VPC, Subnet, EKS, RDS, NAT Gateway). This limitation prevents users from designing comprehensive cloud architectures that include security components, load balancing, serverless computing, storage services, and monitoring infrastructure.

### 1.2 Solution Overview

Build 20 new AWS service node components organized into 5 categories (Security, Networking, Compute, Storage, Monitoring) that integrate with ReactFlow, support Terraform code generation, and provide real-time validation and cost estimation.

### 1.3 Business Value

- **Expanded Coverage**: Support 80%+ of common AWS architecture patterns
- **Faster Design**: Drag-and-drop visual design vs manual Terraform coding
- **Reduced Errors**: Visual validation prevents misconfigurations
- **Cost Visibility**: Real-time cost estimation during design phase

## 3. Functional Requirements

### 3.1 Base Node Component (FR-001)

**Priority**: P0 - Critical

All node components MUST extend a common BaseNode component.

#### FR-001.1 Visual Requirements
| Requirement | Description |
|-------------|-------------|
| FR-001.1.1 | Each node displays an AWS service icon (from AWS architecture icons) |
| FR-001.1.2 | Nodes are color-coded by category (Security=Red, Network=Blue, Compute=Orange, Storage=Green, Monitoring=Purple) |
| FR-001.1.3 | Selected nodes show a highlight border and resize handles |
| FR-001.1.4 | Nodes display a status indicator (configured/unconfigured/error) |
| FR-001.1.5 | Nodes show cost estimate tooltip on hover |

#### FR-001.2 Interaction Requirements
| Requirement | Description |
|-------------|-------------|
| FR-001.2.1 | Nodes can be dragged from palette onto canvas |
| FR-001.2.2 | Nodes support resize by dragging corners |
| FR-001.2.3 | Double-click opens properties panel |
| FR-001.2.4 | Right-click shows context menu (Edit, Duplicate, Delete, View Terraform) |
| FR-001.2.5 | Keyboard shortcuts: Delete key removes node, Ctrl+D duplicates |

#### FR-001.3 Connection Requirements
| Requirement | Description |
|-------------|-------------|
| FR-001.3.1 | Nodes have typed connection handles (input/output) |
| FR-001.3.2 | Invalid connections are rejected with visual feedback |
| FR-001.3.3 | Valid connections snap into place with animation |
| FR-001.3.4 | Connection lines show data flow direction |

**Acceptance Criteria**:
```gherkin
Given a node component on the canvas
When the user hovers over the node
Then a tooltip displays the node name and estimated monthly cost

Given a SecurityGroupNode on the canvas
When the user drags a connection to an EC2InstanceNode
Then the connection is accepted and renders a security group association line

Given a SecurityGroupNode on the canvas
When the user drags a connection to a Route53Node (invalid)
Then the connection is rejected with red visual feedback and error message
```
