## 1. Executive Summary

### 1.1 Problem Statement
Infrastructure teams managing cloud resources across multiple environments (development, staging, production) lack a unified interface to:
- Visualize configuration differences between environments
- Safely promote configurations through the environment pipeline
- Manage environment-specific variables and secrets
- Track and audit changes to production configurations

### 1.2 Business Value
- **Reduced Deployment Errors**: Visual diff prevents configuration drift
- **Faster Time to Production**: Streamlined promotion workflow
- **Improved Compliance**: Audit trail for all production changes
- **Cost Optimization**: Environment-specific resource sizing

### 1.3 Success Metrics
| Metric | Target | Measurement |
|--------|--------|-------------|
| Configuration promotion time | < 5 minutes | Average time from dev to prod |
| Configuration errors | 0 critical | Production incidents from config drift |
| User adoption | > 80% | Teams using environment management UI |
| Audit compliance | 100% | All prod changes have audit trail |

  id: string;
  name: Environment;
  displayName: string;

  id: string;

interface PromotionRecord {
  id: string;
  sourceEnvironment: Environment;
  targetEnvironment: Environment;
  changes: ConfigChange[];
  status: 'pending' | 'approved' | 'rejected' | 'completed' | 'rolled_back';
  requestedBy: string;

  completedAt?: Date;
}

## 7. Integration Points

### 7.1 API Endpoints Required

|----------|--------|---------|
| `/api/environments` | GET | List all environments |
| `/api/environments/:id` | GET | Get environment config |
| `/api/environments/:id` | PUT | Update environment config |
| `/api/environments/:id/variables` | GET/POST | Manage variables |
| `/api/environments/diff` | POST | Calculate diff between envs |
| `/api/environments/promote` | POST | Initiate promotion |
| `/api/environments/promotions` | GET | Promotion history |
| `/api/environments/promotions/:id/approve` | POST | Approve promotion |
| `/api/environments/promotions/:id/rollback` | POST | Rollback promotion |

### 7.2 External Services
- **AWS Secrets Manager**: Secret storage and retrieval
- **Terraform Cloud**: Workspace management per environment
- **Audit Service**: Change tracking and compliance

## 8. Constraints and Assumptions

### 8.1 Constraints
- Production environment cannot be deleted
- At least one environment (dev) must always exist
- Secrets cannot be displayed in plaintext after initial entry
- Promotion to production requires at least one approval

### 8.2 Assumptions
- Users are authenticated and authorized via existing auth system
- AWS credentials are configured for Secrets Manager access
- Terraform workspaces exist for each environment
- Database schema supports multi-environment (confirmed in schema.prisma)
