## 1. Executive Summary

This document defines the architecture for integrating all Visual Designer components into a unified, cohesive interface. The design follows React best practices with emphasis on performance, maintainability, and user experience.

### Key Architectural Decisions
1. **Component Composition** - Modular panels with clear boundaries
2. **State Management** - Extended DesignWizardContext with ReactFlow state
3. **Performance** - Lazy loading, memoization, virtual rendering
4. **Accessibility** - Full keyboard navigation support

### 4.3 Undo/Redo Implementation

```typescript
interface HistoryState {
  past: DesignData[];
  present: DesignData;
  future: DesignData[];
}

// Add to DesignWizardContext
const MAX_HISTORY = 50;

function useUndoRedo() {
  const [history, setHistory] = useState<HistoryState>({
    past: [],
    present: initialDesignData,
    future: [],
  });

  const canUndo = history.past.length > 0;
  const canRedo = history.future.length > 0;

  const undo = useCallback(() => {
    if (!canUndo) return;
    const previous = history.past[history.past.length - 1];
    const newPast = history.past.slice(0, -1);
    setHistory({
      past: newPast,
      present: previous,
      future: [history.present, ...history.future],
    });
  }, [history, canUndo]);

  const redo = useCallback(() => {
    if (!canRedo) return;
    const next = history.future[0];
    const newFuture = history.future.slice(1);
    setHistory({
      past: [...history.past, history.present],
      present: next,
      future: newFuture,
    });
  }, [history, canRedo]);

  const pushState = useCallback((newState: DesignData) => {
    setHistory({
      past: [...history.past.slice(-MAX_HISTORY + 1), history.present],
      present: newState,
      future: [],
    });
  }, [history.present]);

  return { history, canUndo, canRedo, undo, redo, pushState };
}
```

## 7. Keyboard Shortcut Architecture

```typescript
// hooks/useKeyboardShortcuts.ts
interface KeyboardShortcut {
  key: string;
  ctrl?: boolean;
  shift?: boolean;
  alt?: boolean;
  action: () => void;
  description: string;
}

function useKeyboardShortcuts(shortcuts: KeyboardShortcut[]) {
  useEffect(() => {
    const handleKeyDown = (event: KeyboardEvent) => {
      const shortcut = shortcuts.find(s =>
        s.key.toLowerCase() === event.key.toLowerCase() &&
        !!s.ctrl === (event.ctrlKey || event.metaKey) &&
        !!s.shift === event.shiftKey &&
        !!s.alt === event.altKey
      );

      if (shortcut) {
        event.preventDefault();
        shortcut.action();
      }
    };

    window.addEventListener('keydown', handleKeyDown);
    return () => window.removeEventListener('keydown', handleKeyDown);
  }, [shortcuts]);
}

// Usage in VisualDesigner
const shortcuts: KeyboardShortcut[] = [
```

## 10. Architecture Decision Records

### ADR-001: ReactFlow v12 for Canvas

**Status**: Accepted

**Context**: Need interactive canvas for infrastructure visualization.

**Decision**: Use @xyflow/react v12 (ReactFlow).

**Consequences**:
- (+) Rich feature set (nodes, edges, handles, zoom, pan)
- (+) Excellent performance with virtualization
- (+) Active community and maintenance
- (-) Additional 80KB bundle size
- (-) Learning curve for custom nodes

### ADR-002: Extended Context vs. Separate Store

**Status**: Accepted

**Context**: Need undo/redo and additional canvas state.

**Decision**: Extend existing DesignWizardContext rather than adding Zustand.

**Consequences**:
- (+) Single source of truth
- (+) No additional dependencies
- (+) Consistent with existing architecture
- (-) Context may become large
- (-) Potential re-render issues (mitigated with selective hooks)

### ADR-003: Lazy Loading Strategy

**Status**: Accepted

**Context**: Bundle size and initial load time concerns.

**Decision**: Lazy load wizard, templates, and deployment panels.

**Consequences**:
- (+) Faster initial load
- (+) Code splitting benefits
- (-) Loading states needed
- (-) Potential layout shift
