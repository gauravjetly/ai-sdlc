**Dependency Logic**:
```typescript
const LAYER_DEPENDENCIES: Record<LayerType, LayerType[]> = {
  network: [],
  platform: ['network'],
  devops: ['network', 'platform'],
  fullstack: ['network', 'platform', 'devops'],
};

function canSelectLayer(
  layer: LayerType,
  layers: Record<LayerType, { status: LayerStatus }>
): { allowed: boolean; reason?: string } {
  const deps = LAYER_DEPENDENCIES[layer];
  for (const dep of deps) {
    if (layers[dep].status !== 'complete' && layers[dep].status !== 'deployed') {
      return {
        allowed: false,
        reason: `${dep} layer must be completed first`,
      };
    }
  }
  return { allowed: true };
}
```

```typescript
function useDeploymentWebSocket(workflowId: string) {
```

### 3.2 Custom Hooks

**useLayerManagement Hook**:
```typescript
function useLayerManagement() {
  const context = useDesignWizard();

  const canDeployLayer = useCallback((layer: LayerType): CanDeployResult => {
    // Check dependencies
    const deps = LAYER_DEPENDENCIES[layer];
    for (const dep of deps) {
      const depStatus = context.layers[dep].status;
      if (depStatus !== 'deployed') {
        return {
          canDeploy: false,
          reason: `${dep} layer must be deployed first`,
          missingDependencies: [dep],
        };
      }
    }

    // Check if layer is configured
    if (!context.layers[layer].data || context.layers[layer].data.nodes.length === 0) {
      return {
        canDeploy: false,
        reason: 'Layer has no configuration',
      };
    }

    return { canDeploy: true };
  }, [context.layers]);

  const deployLayer = useCallback(async (
    layer: LayerType,
    environment: Environment
  ): Promise<DeploymentResult> => {
    // Implementation
  }, []);

  const rollbackLayer = useCallback(async (
    layer: LayerType
  ): Promise<RollbackResult> => {
    // Check for dependent layers that are deployed
    // Implementation
  }, []);

  return {
    canDeployLayer,
    deployLayer,
    rollbackLayer,
    getLayerDependencies: (layer: LayerType) => LAYER_DEPENDENCIES[layer],
  };
}
```

**useDeploymentLogs Hook**:
```typescript
function useDeploymentLogs(layer: LayerType, maxLines: number = 1000) {
  const [logs, setLogs] = useState<LogEntry[]>([]);
  const logsRef = useRef<HTMLDivElement>(null);

  const appendLog = useCallback((log: LogEntry) => {
    setLogs((prev) => {
      const updated = [...prev, log];
      // Keep only last maxLines
      return updated.slice(-maxLines);
    });
  }, [maxLines]);

  const clearLogs = useCallback(() => {
    setLogs([]);
  }, []);

  // Auto-scroll to bottom
  useEffect(() => {
    if (logsRef.current) {
      logsRef.current.scrollTop = logsRef.current.scrollHeight;
    }
  }, [logs]);

  return { logs, appendLog, clearLogs, logsRef };
}
```

## 6. Architecture Decision Records

### ADR-001: WebSocket for Real-time Deployment Updates

**Status**: Accepted

**Context**: Deployment logs can generate hundreds of lines per second. Users need real-time feedback without excessive polling.

**Decision**: Use WebSocket connection for deployment log streaming.

**Consequences**:
- (+) Real-time updates with low latency
- (+) Reduced server load vs polling
- (-) Need to handle WebSocket reconnection
- (-) More complex error handling

**Alternatives Considered**:
- Server-Sent Events: Simpler but one-way
- Long polling: Higher latency, more server load

---

### ADR-002: SVG-based Dependency Graph

**Status**: Accepted

**Context**: The dependency graph has only 3 nodes and 2 edges. Full graph library may be overkill.

**Decision**: Use custom SVG rendering for the dependency graph instead of a library like ReactFlow.

**Consequences**:
- (+) Smaller bundle size
- (+) Full control over styling and animations
- (+) Simpler for fixed layout
- (-) Need to implement basic interactions manually

**Alternatives Considered**:
- ReactFlow: Already used elsewhere, but heavyweight for 3 nodes
- D3.js: Too complex for simple graph

---

### ADR-003: Context Extension vs Separate Provider

**Status**: Accepted

**Context**: Layer management state could be separate or integrated with existing DesignWizardContext.

**Decision**: Extend existing DesignWizardContext with layer management state.

**Consequences**:
- (+) Single source of truth
- (+) Easier state synchronization
- (+) Consistent with existing patterns
- (-) Context may become large
- Mitigation: Use selective hooks to minimize re-renders

## 7. Security Considerations

### 7.1 Deployment Authorization

```typescript
// Before any deployment action
async function checkDeploymentPermission(
  userId: string,
  layer: LayerType,
  environment: Environment
): Promise<boolean> {
  // Check user has deployment permission for this environment
  const permissions = await getUserPermissions(userId);

  if (environment === 'prod') {
    return permissions.includes('deploy:production');

  return permissions.includes('deploy:non-production');
}
```
