## 1. Executive Summary

This document defines the architecture for the Infrastructure Designer Module of the Catalyst Cloud Management Platform. The design follows a layered architecture pattern with clear separation of concerns, enabling wizard-driven infrastructure design, template management, and Terraform code generation.

### Key Architectural Decisions
1. **Wizard-as-Overlay Pattern**: Wizard drawer wraps existing visual designer
2. **Layer-Based Design**: Network -> Platform -> DevOps with dependency graph
3. **React Context for State**: Centralized state management without Redux complexity
4. **Backend-Driven Validation**: Critical validation logic on server side
5. **Module-Per-Layer Terraform**: Generated code organized by layer

### 8.2 Validation Service

```typescript
class ValidationService {
  async validateDesign(design: Design): Promise<ValidationResult> {
    const errors: ValidationError[] = [];
    const warnings: ValidationWarning[] = [];

    // Layer 1: Schema
    errors.push(...this.validateSchema(design));

    // Layer 2: Business Rules
    errors.push(...this.validateBusinessRules(design));

    // Layer 3: Dependencies
    errors.push(...this.validateDependencies(design));

    // Layer 4: AWS Limits
    errors.push(...await this.validateAWSLimits(design));

    // Layer 5: Security
    warnings.push(...this.validateSecurity(design));

    return {
      valid: errors.length === 0,
      errors,
      warnings,
    };
  }

  private validateCIDR(design: Design): ValidationError[] {
    const errors: ValidationError[] = [];
    const cidrs = this.extractCIDRs(design);

    // Check for overlaps
    for (let i = 0; i < cidrs.length; i++) {
      for (let j = i + 1; j < cidrs.length; j++) {
        if (this.cidrsOverlap(cidrs[i], cidrs[j])) {
          errors.push({
            code: 'CIDR_OVERLAP',
            message: `CIDR ${cidrs[i].value} overlaps with ${cidrs[j].value}`,
            path: cidrs[i].path,
            severity: 'error',
          });
        }
      }
    }

    return errors;
  }
}
```

### 9.2 AWS Credentials Security

```typescript
// NEVER store raw credentials in database
// Use AWS IAM roles where possible

class AWSCredentialService {
  // For local development: use environment variables
  getCredentials(): AWSCredentials {
    return {
      accessKeyId: process.env.AWS_ACCESS_KEY_ID,
      secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY,
      region: process.env.AWS_REGION,
    };
  }

  // For production: use IAM roles attached to compute
  async assumeRole(roleArn: string): Promise<AWSCredentials> {
    const sts = new STSClient({});
    const response = await sts.send(new AssumeRoleCommand({
      RoleArn: roleArn,
      RoleSessionName: 'CatalystDesigner',
      DurationSeconds: 3600,
    }));
    return response.Credentials;
  }
}
```
