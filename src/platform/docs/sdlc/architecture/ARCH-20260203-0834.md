## 1. Executive Summary

This document defines the architecture for the Terraform Module Generator system. The system converts visual designer node configurations into production-ready Terraform HCL code using a modular, extensible generator framework.

### 1.1 Architecture Principles

1. **Single Responsibility**: Each generator handles exactly one AWS resource type
2. **Open/Closed**: New generators can be added without modifying existing code
3. **Dependency Injection**: Generators receive dependencies rather than creating them
4. **Composition over Inheritance**: Use mixins and composition for shared behavior
5. **Fail Fast**: Validate inputs early and provide clear error messages

### 1.2 Key Decisions

- **Strategy Pattern** for generator selection based on node type
- **Template Method Pattern** for consistent generation workflow
- **Builder Pattern** for constructing complex HCL structures
- **Registry Pattern** for generator lookup and instantiation

## 4. Class Design

### 4.1 Core Classes

#### 4.1.1 BaseGenerator (Abstract)

```typescript
/**
 * Abstract base class for all Terraform generators
 * Implements Template Method pattern for consistent generation workflow
 */
abstract class BaseGenerator<T extends BaseNodeData> {
  // Abstract properties - must be defined by subclasses
  abstract readonly terraformResourceType: string;
  abstract readonly nodeServiceType: AWSServiceType;
  abstract readonly category: NodeCategory;

  // Template method - defines the generation workflow
  generate(node: T, context: GeneratorContext): GeneratorOutput {
    this.validateNode(node);
    const dependencies = this.resolveDependencies(node, context);

    return {
      mainTf: this.generateMainTF(node, context),
      variablesTf: this.generateVariablesTF(node, context),
      outputsTf: this.generateOutputsTF(node, context),
      versionsTf: this.generateVersionsTF(),
      dependencies,
      metadata: this.getMetadata(node),
    };
  }

  // Abstract methods - must be implemented by subclasses
  protected abstract generateMainTF(node: T, context: GeneratorContext): string;
  protected abstract generateVariablesTF(node: T, context: GeneratorContext): string;
  protected abstract generateOutputsTF(node: T, context: GeneratorContext): string;
  protected abstract getResourceOutputs(): ResourceOutput[];

  // Common methods - inherited by all generators
  protected validateNode(node: T): void { ... }
  protected resolveDependencies(node: T, context: GeneratorContext): string[] { ... }
  protected formatTags(tags: Record<string, string>, context: GeneratorContext): string { ... }
  protected generateVersionsTF(): string { ... }
  protected sanitizeResourceName(name: string): string { ... }
  protected getMetadata(node: T): GeneratorMetadata { ... }

  // Utility methods
  protected buildHCL(): HCLBuilder { return new HCLBuilder(); }
  protected buildVariable(): VariableBuilder { return new VariableBuilder(); }
  protected buildOutput(): OutputBuilder { return new OutputBuilder(); }
}
```

### 4.2 Generator Implementation Pattern

Each generator follows this pattern:

```typescript
/**
 * Example: VPC Generator
 */
class VPCGenerator extends BaseGenerator<VPCNodeData> {
  readonly terraformResourceType = 'aws_vpc';
  readonly nodeServiceType: AWSServiceType = 'vpc';
  readonly category: NodeCategory = 'network';

  protected generateMainTF(node: VPCNodeData, context: GeneratorContext): string {
    const hcl = this.buildHCL();

    hcl.resource('aws_vpc', this.sanitizeResourceName(node.name))
      .attribute('cidr_block', node.cidrBlock)
      .attribute('enable_dns_hostnames', node.enableDnsHostnames ?? true)
      .attribute('enable_dns_support', node.enableDnsSupport ?? true)
      .tags(this.formatTags(node.tags, context));

    return hcl.toString();
  }

  protected generateVariablesTF(node: VPCNodeData, context: GeneratorContext): string {
    const vars = this.buildVariable();

    vars.add('cidr_block', {
      type: 'string',
      description: 'CIDR block for the VPC',
      default: node.cidrBlock,
    });

    vars.add('enable_dns_hostnames', {
      type: 'bool',
      description: 'Enable DNS hostnames in the VPC',
      default: node.enableDnsHostnames ?? true,
    });

    return vars.toString();
  }

  protected generateOutputsTF(node: VPCNodeData, context: GeneratorContext): string {
    const outputs = this.buildOutput();
    const resourceName = this.sanitizeResourceName(node.name);

    outputs.add('vpc_id', {
      description: 'ID of the VPC',
      value: `aws_vpc.${resourceName}.id`,
    });

    outputs.add('vpc_cidr_block', {
      description: 'CIDR block of the VPC',
      value: `aws_vpc.${resourceName}.cidr_block`,
    });

    return outputs.toString();
  }

  protected getResourceOutputs(): ResourceOutput[] {
    return [
      { name: 'vpc_id', type: 'string', description: 'VPC ID' },
      { name: 'vpc_cidr_block', type: 'string', description: 'VPC CIDR block' },
    ];
  }
}
```

### 5.2 Dependency Resolution Algorithm

```typescript
class DependencyResolver {
  resolve(nodes: VisualDesignerNode[], edges: Connection[]): SortedNodes {
    // 1. Build adjacency list
    const graph = this.buildDependencyGraph(nodes, edges);

    // 2. Detect cycles
    if (this.hasCycle(graph)) {
      throw new CircularDependencyError(this.findCycle(graph));
    }

    // 3. Topological sort using Kahn's algorithm
    const sorted = this.kahnSort(graph);

    // 4. Apply layer ordering for same-level nodes
    return this.applyLayerOrdering(sorted);
  }

  private layerOrder: NodeCategory[] = [
    'network',     // VPCs, Subnets, Gateways
    'security',    // Security Groups, IAM
    'compute',     // EC2, EKS, Lambda
    'storage',     // S3, DynamoDB, EFS
    'networking',  // Load Balancers, Route53
    'monitoring',  // CloudWatch, SNS, SQS
  ];
}
```

### 7.3 No Hardcoded Secrets Rule

```typescript
class SecurityValidator {
  validateNoHardcodedSecrets(hcl: string): ValidationResult {
    const secretPatterns = [
      /AKIA[0-9A-Z]{16}/,           // AWS Access Key
      /password\s*=\s*"[^"]+"/i,    // Hardcoded password
      /secret\s*=\s*"[^"]+"/i,      // Hardcoded secret
      /-----BEGIN (?:RSA )?PRIVATE KEY-----/, // Private keys
    ];

    for (const pattern of secretPatterns) {
      if (pattern.test(hcl)) {
        return { valid: false, error: 'Hardcoded secret detected' };
      }
    }
    return { valid: true };
  }
}
```

### 8.2 Error Recovery

```typescript
class GeneratorOrchestrator {
  async generate(input: GeneratorInput): Promise<GeneratorResult> {
    const errors: GeneratorError[] = [];
    const modules: ModuleInfo[] = [];

    for (const node of sortedNodes) {
      try {
        const module = await this.generateModule(node, context);
        modules.push(module);
      } catch (error) {
        if (error instanceof GeneratorError) {
          errors.push(error);
          // Continue with next node (partial success)
        } else {
          throw error; // Unknown error, fail fast
        }
      }
    }

    return {
      success: errors.length === 0,
      modules,
      errors,
    };
  }
}
```

## 9. Performance Considerations

### 9.1 Performance Requirements

| Metric | Requirement | Implementation |
|--------|-------------|----------------|
| 100-node generation | < 5 seconds | Parallel generation where possible |
| Memory usage | < 500MB | Stream large outputs |
| Single node preview | < 100ms | Cached template compilation |

### 9.2 Optimization Strategies

1. **Template Caching**: Pre-compile HCL templates at startup
2. **Parallel Generation**: Generate independent modules in parallel
3. **Lazy Loading**: Load generator modules on demand
4. **String Building**: Use StringBuilder pattern for HCL construction

```typescript
class GeneratorOrchestrator {
  async generate(input: GeneratorInput): Promise<GeneratorResult> {
    // Sort nodes by dependency
    const sortedNodes = this.dependencyResolver.resolve(input.nodes, input.edges);

    // Group independent nodes for parallel processing
    const batches = this.groupByDependencyLevel(sortedNodes);

    for (const batch of batches) {
      // Generate batch in parallel
      await Promise.all(batch.map(node => this.generateModule(node)));
    }
  }
}
```
