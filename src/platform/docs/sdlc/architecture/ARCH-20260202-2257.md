## 1. Executive Summary

This document defines the architecture for the Template System UI, a set of React components that enable users to browse, preview, create, edit, and apply infrastructure templates within the Infrastructure Designer module.

### Key Architectural Decisions
- **Component-based architecture** with clear separation of concerns
- **Custom hooks** for data fetching and state management
- **React Query** for server state management with caching
- **URL-based state** for shareable filter configurations
- **Lazy loading** for performance optimization
- **Accessibility-first** design with ARIA compliance

## 5. API Integration Layer

### 5.1 API Service

```typescript
// utils/templateApi.ts

import { apiClient } from '@/services/api';
import type {
  DesignTemplate,
  TemplateListParams,
  TemplateListResponse,
  CreateTemplateDTO,
  UpdateTemplateDTO,
  CloneTemplateDTO,
  TemplateVersion,
} from '../types/template.types';

const BASE_URL = '/api/v1/templates';

export const templateApi = {
  /**
   * List templates with filtering, pagination, and sorting
   */
  async list(params: TemplateListParams): Promise<TemplateListResponse> {
    const response = await apiClient.get(BASE_URL, { params });
    return response.data;
  },

  /**
   * Get single template by ID
   */
  async get(id: string): Promise<DesignTemplate> {
    const response = await apiClient.get(`${BASE_URL}/${id}`);
    return response.data.data;
  },

  /**
   * Create new template
   */
  async create(data: CreateTemplateDTO): Promise<DesignTemplate> {
    const response = await apiClient.post(BASE_URL, data);
    return response.data.data;
  },

  /**
   * Update existing template
   */
  async update(id: string, data: UpdateTemplateDTO): Promise<DesignTemplate> {
    const response = await apiClient.put(`${BASE_URL}/${id}`, data);
    return response.data.data;
  },

  /**
   * Delete template
   */
  async delete(id: string): Promise<void> {
    await apiClient.delete(`${BASE_URL}/${id}`);
  },

  /**
   * Get template version history
   */
  async getVersions(id: string): Promise<TemplateVersion[]> {
    const response = await apiClient.get(`${BASE_URL}/${id}/versions`);
    return response.data.data;
  },

  /**
   * Clone template
   */
  async clone(id: string, data: CloneTemplateDTO): Promise<DesignTemplate> {
    const response = await apiClient.post(`${BASE_URL}/${id}/clone`, data);
    return response.data.data;
  },

  /**
   * Upload thumbnail (future endpoint)
   */
  async uploadThumbnail(id: string, file: File): Promise<string> {
    const formData = new FormData();
    formData.append('thumbnail', file);
    const response = await apiClient.post(
      `${BASE_URL}/${id}/thumbnail`,
      formData,
      { headers: { 'Content-Type': 'multipart/form-data' } }
    );
    return response.data.data.url;
  },
};
```

### 5.2 React Query Hooks

```typescript
// hooks/useTemplates.ts

import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import { templateApi } from '../utils/templateApi';
import type { TemplateListParams } from '../types/template.types';

// Query keys
export const templateKeys = {
  all: ['templates'] as const,
  lists: () => [...templateKeys.all, 'list'] as const,
  list: (params: TemplateListParams) => [...templateKeys.lists(), params] as const,
  details: () => [...templateKeys.all, 'detail'] as const,
  detail: (id: string) => [...templateKeys.details(), id] as const,
  versions: (id: string) => [...templateKeys.detail(id), 'versions'] as const,
};

/**
 * Hook for fetching template list with caching
 */
export function useTemplates(params: TemplateListParams) {
  return useQuery({
    queryKey: templateKeys.list(params),
    queryFn: () => templateApi.list(params),
    staleTime: 30 * 1000, // 30 seconds
    keepPreviousData: true, // Smooth pagination
  });
}

/**
 * Hook for fetching single template
 */
export function useTemplate(id: string, enabled = true) {
  return useQuery({
    queryKey: templateKeys.detail(id),
    queryFn: () => templateApi.get(id),
    enabled: enabled && !!id,
    staleTime: 60 * 1000, // 1 minute
  });
}

/**
 * Hook for fetching template versions
 */
export function useTemplateVersions(id: string, enabled = true) {
  return useQuery({
    queryKey: templateKeys.versions(id),
    queryFn: () => templateApi.getVersions(id),
    enabled: enabled && !!id,
  });
}

/**
 * Hook for template mutations (create, update, delete, clone)
 */
export function useTemplateMutations() {
  const queryClient = useQueryClient();

  const createMutation = useMutation({
    mutationFn: templateApi.create,
    onSuccess: () => {
      queryClient.invalidateQueries(templateKeys.lists());
    },
  });

  const updateMutation = useMutation({
    mutationFn: ({ id, data }: { id: string; data: any }) =>
      templateApi.update(id, data),
    onSuccess: (_, { id }) => {
      queryClient.invalidateQueries(templateKeys.detail(id));
      queryClient.invalidateQueries(templateKeys.lists());
    },
  });

  const deleteMutation = useMutation({
    mutationFn: templateApi.delete,
    onSuccess: () => {
      queryClient.invalidateQueries(templateKeys.lists());
    },
  });

  const cloneMutation = useMutation({
    mutationFn: ({ id, data }: { id: string; data: any }) =>
      templateApi.clone(id, data),
    onSuccess: () => {
      queryClient.invalidateQueries(templateKeys.lists());
    },
  });

  return {
    create: createMutation,
    update: updateMutation,
    delete: deleteMutation,
    clone: cloneMutation,
  };
}
```

## 9. Error Handling Architecture

### 9.1 Error Boundary Structure

```typescript
// Error boundary for template components
class TemplateErrorBoundary extends React.Component {
  state = { hasError: false, error: null };

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  render() {
    if (this.state.hasError) {
      return (
        <TemplateErrorFallback
          error={this.state.error}
          onRetry={() => this.setState({ hasError: false })}
        />
      );
    }
    return this.props.children;
  }
}
```

### 9.2 API Error Handling

```typescript
// Centralized error handling
export function handleTemplateError(error: unknown): TemplateError {
  if (axios.isAxiosError(error)) {
    const status = error.response?.status;
    const message = error.response?.data?.error?.message;

    switch (status) {
      case 401:
        return { code: 'UNAUTHORIZED', message: 'Please log in to continue' };
      case 403:
        return { code: 'FORBIDDEN', message: 'You do not have permission' };
      case 404:
        return { code: 'NOT_FOUND', message: 'Template not found' };
      case 422:
        return { code: 'VALIDATION', message: message || 'Invalid data' };
      case 429:
        return { code: 'RATE_LIMITED', message: 'Too many requests, please wait' };
      default:
        return { code: 'SERVER_ERROR', message: 'Something went wrong' };
    }
  }
  return { code: 'UNKNOWN', message: 'An unexpected error occurred' };
}
```

```typescript
// testUtils/renderWithProviders.tsx
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { render } from '@testing-library/react';
import { MemoryRouter } from 'react-router-dom';

export function renderWithProviders(
  ui: React.ReactElement,
  { route = '/', ...options } = {}
) {
  const queryClient = new QueryClient({
    defaultOptions: { queries: { retry: false } },
  });

  return render(
    <QueryClientProvider client={queryClient}>
      <MemoryRouter initialEntries={[route]}>
        {ui}
      </MemoryRouter>
    </QueryClientProvider>,
    options
  );
}
```

## 12. Architecture Decision Records

### ADR-001: React Query for Server State

**Context**: Need efficient data fetching with caching for template lists.

**Decision**: Use React Query (TanStack Query) for all server state management.

**Rationale**:
- Built-in caching reduces API calls
- Automatic background refetching keeps data fresh
- keepPreviousData enables smooth pagination
- Devtools for debugging

**Consequences**:
- Additional dependency (~12KB gzipped)
- Learning curve for team
- Excellent DX with built-in loading/error states

### ADR-002: URL-Based Filter State

**Context**: Users want to share filtered views and maintain state on refresh.

**Decision**: Encode filter state in URL search parameters.

**Rationale**:
- Shareable URLs for collaboration
- Browser back/forward navigation works
- Persists across page refresh
- SEO-friendly for public templates

**Consequences**:
- More complex state synchronization
- URL can become long with many filters
- Need to validate URL params

### ADR-003: Lazy Loading for Performance

**Context**: Template preview and editor are heavy components.

**Decision**: Use React.lazy and Suspense for code splitting.

**Rationale**:
- Reduces initial bundle size by ~40%
- Preview/editor loaded only when needed
- Suspense provides smooth loading states

**Consequences**:
- Slight delay on first open
- Need fallback components
- More complex error handling
