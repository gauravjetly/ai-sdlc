## 🔧 Future Configuration

### For New Projects

**Option 1: Auto-Configure (Recommended)**
All new SDLC runs will be auto-detected as AI-SDLC features if they:
- Have GitHub URLs pointing to the repo
- Mention "AI-SDLC" or "AISDLC" in description
- Have files in `src/` directory

**Option 2: Use Configuration Tool**
```bash
cd ~/aisdlc-2.1.0
./scripts/configure-project-mapping.sh --configure \
  <new-project-id> \
  "AI-SDLC Framework" \
  "https://github.com/DLTKEngineering/aisdlc" \
  "New Feature Name"
```

**Option 3: Bulk Re-Configure**
```bash
# Re-scan and configure all projects
./scripts/configure-project-mapping.sh --bulk aisdlc
```

### For Different Main Projects

If you start tracking features for other projects (e.g., Claude-Admin):

```bash
# Add to registry
vim ~/.claude/sdlc-registry/project-mapping.json
# Add project entry

# Configure specific projects
./scripts/configure-project-mapping.sh --configure \
  <project-id> \
  "Claude Admin" \
  "https://github.com/org/claude-admin" \
  "Feature Name"
```
