## 1. Executive Summary

This document describes the architecture for a RESTful Task Management API that provides CRUD operations for tasks with user authentication, authorization, and comprehensive observability. The architecture prioritizes simplicity, maintainability, security, and performance.

### 1.1 Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Runtime | Node.js 20 LTS | Performance, ecosystem, async I/O |
| Framework | Express.js | Mature, flexible, widely adopted |
| Language | TypeScript | Type safety, better tooling |
| Database | PostgreSQL | ACID compliance, JSON support, reliability |
| ORM | Prisma | Type-safe queries, migrations, excellent DX |
| Authentication | JWT (RS256) | Stateless, secure, industry standard |
| Containerization | Docker | Portability, consistency |
| Testing | Jest + Supertest | Comprehensive testing framework |

## 4. Component Design

### 4.1 Presentation Layer

#### 4.1.1 Controllers

Controllers handle HTTP requests and delegate to application services.

```typescript
// src/presentation/controllers/task.controller.ts
export class TaskController {
  constructor(private taskService: TaskService) {}

  async create(req: Request, res: Response): Promise<void> {
    const task = await this.taskService.create(req.user.id, req.body);
    res.status(201).json({ data: task });
  }

  async findById(req: Request, res: Response): Promise<void> {
    const task = await this.taskService.findById(req.params.id, req.user.id);
    res.status(200).json({ data: task });
  }

  async findAll(req: Request, res: Response): Promise<void> {
    const result = await this.taskService.findAll(req.user.id, req.query);
    res.status(200).json({ data: result.tasks, meta: { pagination: result.pagination } });
  }

  async update(req: Request, res: Response): Promise<void> {
    const task = await this.taskService.update(req.params.id, req.user.id, req.body);
    res.status(200).json({ data: task });
  }

  async delete(req: Request, res: Response): Promise<void> {
    await this.taskService.delete(req.params.id, req.user.id);
    res.status(204).send();
  }
}
```

#### 4.1.2 Middleware

```typescript
// Authentication Middleware
export const authenticate = async (req: Request, res: Response, next: NextFunction) => {
  const token = extractBearerToken(req.headers.authorization);
  if (!token) throw new UnauthorizedError('Missing authentication token');

  const payload = await jwtService.verify(token);
  req.user = { id: payload.sub, email: payload.email };
  next();
};

// Error Handling Middleware
export const errorHandler = (err: Error, req: Request, res: Response, next: NextFunction) => {
  const statusCode = err instanceof AppError ? err.statusCode : 500;
  const errorCode = err instanceof AppError ? err.code : 'INTERNAL_ERROR';

  logger.error({ err, requestId: req.id }, 'Request failed');

  res.status(statusCode).json({
    error: {
      code: errorCode,
      message: err.message,
      ...(err instanceof ValidationError && { details: err.details })
    },
    meta: { timestamp: new Date().toISOString(), requestId: req.id }
  });
};
```

### 4.2 Application Layer

#### 4.2.1 Services

Services orchestrate use cases and enforce business rules.

```typescript
// src/application/services/task.service.ts
export class TaskService {
  constructor(private taskRepository: ITaskRepository) {}

  async create(userId: string, dto: CreateTaskDto): Promise<Task> {
    const task = Task.create({
      userId,
      title: dto.title,
      description: dto.description,
      status: dto.status || TaskStatus.TODO,
      priority: dto.priority || Priority.MEDIUM,
      dueDate: dto.dueDate ? new Date(dto.dueDate) : null
    });

    return this.taskRepository.save(task);
  }

  async findById(taskId: string, userId: string): Promise<Task> {
    const task = await this.taskRepository.findById(taskId);
    if (!task) throw new NotFoundError('Task', taskId);
    if (task.userId !== userId) throw new ForbiddenError('Access denied');
    return task;
  }

  async findAll(userId: string, query: TaskQueryDto): Promise<PaginatedResult<Task>> {
    return this.taskRepository.findByUser(userId, {
      status: query.status,
      priority: query.priority,
      page: query.page || 1,
      pageSize: Math.min(query.pageSize || 20, 100),
      sortBy: query.sortBy || 'createdAt',
      order: query.order || 'desc'
    });
  }

  async update(taskId: string, userId: string, dto: UpdateTaskDto): Promise<Task> {
    const task = await this.findById(taskId, userId);
    task.update(dto);
    return this.taskRepository.save(task);
  }

  async delete(taskId: string, userId: string): Promise<void> {
    await this.findById(taskId, userId); // Authorization check
    await this.taskRepository.delete(taskId);
  }
}
```

### 4.3 Domain Layer

#### 4.3.1 Entities

```typescript
// src/domain/entities/task.entity.ts
export class Task {
  readonly id: string;
  readonly userId: string;
  private _title: string;
  private _description: string | null;
  private _status: TaskStatus;
  private _priority: Priority;
  private _dueDate: Date | null;
  readonly createdAt: Date;
  private _updatedAt: Date;

  private constructor(props: TaskProps) {
    this.id = props.id;
    this.userId = props.userId;
    this._title = props.title;
    this._description = props.description;
    this._status = props.status;
    this._priority = props.priority;
    this._dueDate = props.dueDate;
    this.createdAt = props.createdAt;
    this._updatedAt = props.updatedAt;
  }

  static create(props: CreateTaskProps): Task {
    if (!props.title || props.title.trim().length === 0) {
      throw new ValidationError('Title is required');
    }
    if (props.title.length > 200) {
      throw new ValidationError('Title must be 200 characters or less');
    }

    return new Task({
      id: generateUuid(),
      userId: props.userId,
      title: props.title.trim(),
      description: props.description?.trim() || null,
      status: props.status || TaskStatus.TODO,
      priority: props.priority || Priority.MEDIUM,
      dueDate: props.dueDate || null,
      createdAt: new Date(),
      updatedAt: new Date()
    });
  }

  update(props: UpdateTaskProps): void {
    if (props.title !== undefined) {
      if (!props.title || props.title.trim().length === 0) {
        throw new ValidationError('Title cannot be empty');
      }
      this._title = props.title.trim();
    }
    if (props.description !== undefined) {
      this._description = props.description?.trim() || null;
    }
    if (props.status !== undefined) {
      this._status = props.status;
    }
    if (props.priority !== undefined) {
      this._priority = props.priority;
    }
    if (props.dueDate !== undefined) {
      this._dueDate = props.dueDate;
    }
    this._updatedAt = new Date();
  }

  // Getters
  get title(): string { return this._title; }
  get description(): string | null { return this._description; }
  get status(): TaskStatus { return this._status; }
  get priority(): Priority { return this._priority; }
  get dueDate(): Date | null { return this._dueDate; }
  get updatedAt(): Date { return this._updatedAt; }
}
```

#### 4.4.2 Repository Implementation

```typescript
// src/infrastructure/database/repositories/task.repository.ts
export class TaskRepository implements ITaskRepository {
  constructor(private prisma: PrismaClient) {}

  async save(task: Task): Promise<Task> {
    const data = {
      id: task.id,
      userId: task.userId,
      title: task.title,
      description: task.description,
      status: task.status,
      priority: task.priority,
      dueDate: task.dueDate,
      createdAt: task.createdAt,
      updatedAt: task.updatedAt
    };

    await this.prisma.task.upsert({
      where: { id: task.id },
      create: data,
      update: data
    });

    return task;
  }

  async findById(id: string): Promise<Task | null> {
    const record = await this.prisma.task.findUnique({ where: { id } });
    return record ? this.toDomain(record) : null;
  }

  async findByUser(userId: string, options: QueryOptions): Promise<PaginatedResult<Task>> {
    const where = {
      userId,
      ...(options.status && { status: options.status }),
      ...(options.priority && { priority: options.priority })
    };

    const [records, total] = await Promise.all([
      this.prisma.task.findMany({
        where,
        orderBy: { [options.sortBy]: options.order },
        skip: (options.page - 1) * options.pageSize,
        take: options.pageSize
      }),
      this.prisma.task.count({ where })
    ]);

    return {
      tasks: records.map(this.toDomain),
      pagination: {
        page: options.page,
        pageSize: options.pageSize,
        total,
        totalPages: Math.ceil(total / options.pageSize)
      }
    };
  }

  async delete(id: string): Promise<void> {
    await this.prisma.task.delete({ where: { id } });
  }

  private toDomain(record: PrismaTask): Task {
    return Task.fromPersistence({
      id: record.id,
      userId: record.userId,
      title: record.title,
      description: record.description,
      status: record.status as TaskStatus,
      priority: record.priority as Priority,
      dueDate: record.dueDate,
      createdAt: record.createdAt,
      updatedAt: record.updatedAt
    });
  }
}
```

## 5. Security Architecture

### 5.1 Authentication Flow

```
                                                    JWT Access Token
Client                API Gateway               Task API                  Database
  |                       |                         |                         |
  |---(1) POST /login---->|                         |                         |
  |   (email, password)   |                         |                         |
  |                       |----(2) Authenticate---->|                         |
  |                       |                         |---(3) Find User-------->|
  |                       |                         |<---(4) User Record------|
  |                       |                         |                         |
  |                       |                         |---(5) Verify Password   |
  |                       |                         |---(6) Generate JWT      |
  |                       |<---(7) Tokens-----------|                         |
  |<--(8) accessToken,----|                         |                         |
  |      refreshToken     |                         |                         |
  |                       |                         |                         |
  |---(9) GET /tasks----->|                         |                         |
  |   Authorization:      |                         |                         |
  |   Bearer <token>      |                         |                         |
  |                       |---(10) Verify Token---->|                         |
  |                       |<---(11) User Context----|                         |
  |                       |                         |---(12) Query Tasks----->|
  |                       |                         |<---(13) Task Data-------|
  |<--(14) Tasks----------|<---(Tasks)--------------|                         |
```

### 5.2 JWT Token Structure

**Access Token** (expires: 1 hour):
```json
{
  "header": {
    "alg": "RS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "user-uuid",
    "email": "user@example.com",
    "iat": 1705312800,
    "exp": 1705316400,
    "iss": "task-api",
    "aud": "task-api-client"
  }
}
```

**Refresh Token** (expires: 7 days):
- Stored as opaque token
- Hashed in database
- Single-use (rotation on refresh)

### 5.3 Security Controls

| Control | Implementation | Purpose |
|---------|----------------|---------|
| Input Validation | Zod schemas | Prevent injection attacks |
| Rate Limiting | express-rate-limit | Prevent DoS/brute force |
| Password Hashing | bcrypt (cost 12) | Protect credentials |
| JWT Signing | RS256 (asymmetric) | Secure token verification |
| CORS | Configurable origins | Prevent CSRF |
| Helmet | Security headers | Common vulnerability protection |
| SQL Injection | Prisma ORM (parameterized) | Database protection |

## 11. Non-Functional Requirements Implementation

### 11.1 Performance

| Requirement | Implementation |
|-------------|----------------|
| p95 < 200ms | Database indexes, connection pooling, efficient queries |
| p99 < 500ms | Response compression, caching headers |
| 100+ req/sec | Stateless design, horizontal scaling |

### 11.2 Scalability

| Aspect | Implementation |
|--------|----------------|
| Horizontal | Stateless API, no server-side sessions |
| Database | Connection pooling (pgBouncer), read replicas |
| Memory | Stream large responses, limit pagination |

### 11.3 Security

| Requirement | Implementation |
|-------------|----------------|
| Authentication | JWT RS256 with key rotation |
| Authorization | User-scoped data access |
| Encryption | TLS 1.3, bcrypt passwords |
| Input Validation | Zod schemas, sanitization |
| Rate Limiting | Per-user limits |
