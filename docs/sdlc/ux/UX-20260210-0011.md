**Author**: UX Agent

---

## 1. Design Principles

1. **Progressive Disclosure**: Show essential options first, reveal advanced features on demand
2. **Consistency**: Follow existing Visual Designer patterns (MUI components, color scheme, layout)
3. **Immediate Feedback**: Real-time status updates via WebSocket; visual indicators for all states
4. **Accessibility**: WCAG 2.1 AA minimum; keyboard navigation for all interactive elements
5. **Efficiency**: Power users can create schedules in under 30 seconds

---

## 2. Information Architecture

```
Scheduling (New Top-Level Navigation Item)

+-- Dashboard
|   +-- Summary Cards (scheduled, running, completed, failed counts)
|   +-- Timeline View (Gantt-style upcoming work)
|   +-- Agent Status Cards (health, capacity, active work)

+-- Work Items
|   +-- Work Item List (filterable table)

|   +-- Work Item Detail (status, logs, memory, artifacts)
|
+-- Triggers
|   +-- Trigger List

|   +-- Trigger Detail (history, fire count)
|
+-- Templates
|   +-- Template Gallery
|   +-- Create Template
|   +-- Template Detail (apps, dependencies)

+-- Agent Memory
|   +-- Memory Browser (per agent)

```

## 5. Interaction Patterns

### Real-time Updates
- WebSocket connection established on dashboard mount
- Work item status changes reflected within 1 second
- Execution progress bar updates every 5 seconds
- Agent status refreshed every 30 seconds
- Toast notifications for completions and failures

### Loading States
- Skeleton screens for initial load (not spinners)
- Optimistic UI updates for create/pause/resume actions
- Error boundaries with retry options

### Responsive Behavior
- Desktop (>1200px): Full layout with sidebar, timeline, and agent cards
- Tablet (768-1200px): Collapsible sidebar, stacked agent cards
- Mobile (<768px): Bottom navigation, single-column layout, simplified timeline

---

## 6. Accessibility Requirements

- All interactive elements keyboard navigable (Tab order defined)
- ARIA labels on all status indicators (color alone does not convey status)
- Screen reader announcements for real-time status changes
- High contrast mode support via MUI theme
- Focus management in multi-step dialogs (focus returns to trigger on close)
- Minimum touch target size: 44x44px on mobile
