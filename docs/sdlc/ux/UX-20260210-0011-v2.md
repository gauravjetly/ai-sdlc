## 1. Design Philosophy: Insight Density

The v1 dashboard used traditional card/table layouts that consumed excessive vertical space. The v2 dashboard is inspired by modern observability tools (Grafana, Linear, Vercel) and optimizes for **insight density** -- maximum information in minimum viewport area.

### Principles
1. **Glanceable**: Key metrics visible in 3 seconds without scrolling
2. **Compact**: No wasted whitespace; every pixel earns its space
3. **Layered**: Summary on top, drill-down on click
4. **Alive**: Real-time updates via WebSocket; subtle animations show activity
5. **Dark-friendly**: Works equally well in light and dark themes

## 3. Component Specifications

### 3.1 Metrics Strip (48px, always visible)

A single-row horizontal strip with 5 key metrics. Each metric is a compact chip:

```
[count] [label] [trend indicator]
```

| Metric | Format | Color Logic |
|--------|--------|-------------|
| Active Projects | "12 Active" | Blue |
| At Risk | "3 At Risk" | Amber if > 0, green if 0 |
| Agent Utilization | "87% Agents" | Green < 80%, Amber 80-95%, Red > 95% |
| Avg Phase Duration | "2.4d Avg" | Neutral |
| Velocity Trend | "+15%" or "-8%" | Green if positive, Red if negative |

Implementation: Single `<Box display="flex">` with `<Chip>` components. No cards, no padding waste.

### 3.2 Project Pipeline Table (core component)

Each project is a single row in a compact table. The key innovation is the **phase indicator strip** -- 7 small circles/dots in a row showing phase status.

**Phase Indicator Encoding**:
- Completed: Solid filled circle (green)
- In Progress: Pulsing circle (blue) with animated ring
- Failed: Solid filled circle (red)
- Blocked: Solid filled circle (amber)
- Pending: Empty circle (gray outline)
- Skipped: Dash (gray)

```
Column Layout:
| Name (flex) | Pipeline [7 dots] | Priority | Delivery | Health | Actions |

Row height: 40px
Hover: Shows phase names as tooltip over dots
Click: Expands inline detail panel (accordion style)
```

### 3.3 Bottom Analytics Panel (tabbed, collapsible)

Three tabs in a compact panel:

**Tab 1: Agent Pool** -- Horizontal stacked bars per agent type showing utilization.


Each bar is max 200px wide. Label, bar, percentage, queue count all on one line.

## 4. Create Project Dialog (Streamlined)

Single-page form instead of multi-step wizard (faster for power users):

```
+--------------------------------------------------+
| NEW PROJECT                             [X Close] |
|                                                    |
| Name *        [________________________________]   |
| Description   [________________________________]   |
|               [________________________________]   |
|                                                    |
| Delivery Date *  [Feb 20, 2026]  Priority [HIGH v]|
| Start          ( ) Now  ( ) Scheduled [_________] |
| Tags           [auth] [backend] [+ Add]           |
|                                                    |
| Lifecycle: Full SDLC (7 phases)                    |
| Estimated: ~10.3 days based on historical data     |
|                                                    |
|                          [Cancel]  [Create Project]|
+--------------------------------------------------+
```

No agent assignment step needed -- the system auto-allocates from the pool.

## 7. Interaction Patterns

### Keyboard Shortcuts
- `N`: Open new project dialog
- `R`: Refresh dashboard
- `1-9`: Focus project row 1-9
- `Enter`: Expand focused project detail
- `Esc`: Collapse expanded detail / close dialogs

### Micro-animations
- Phase completion: Dot fills with a 200ms scale-up animation
- Phase start: Dot begins pulsing ring animation (CSS keyframes)
- Risk change: Health chip slides color transition (300ms)
- Agent allocation: Brief highlight flash on agent pool bar

### Real-time Updates
- WebSocket connection on mount
- Project phase changes update in <1 second
- Agent pool status updates every 5 seconds
- Metrics strip recalculates on every project state change
