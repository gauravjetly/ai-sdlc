## 5. Rollback Procedure

### Immediate Rollback (< 1 minute)

```bash
# Set environment variables to file-based mode
export STORAGE=file
export EVENT_BUS=file

# Restart the application
# The system will automatically use file-based providers
```

### Full Infrastructure Rollback

```bash
# Stop Docker services
cd src/agent-mesh
docker compose down

# Reset environment
export STORAGE=file
export EVENT_BUS=file
```

No data loss occurs during rollback because:
1. File-based storage continues to work independently
2. The system defaults to file-based mode when external services are unavailable
3. Graceful fallback is built into the initialization code

## 6. Monitoring

### Health Check Endpoint

The `AgentMesh.healthCheck()` method now reports:
- **memoryService**: Provider type (file/postgres), item count
- **eventBus**: Provider type (file/bullmq), subscriber count, published count
- **registry**: Agent count and status
- **messageBus**: Circuit breaker states

### Key Metrics to Monitor

| Metric | Source | Alert Threshold |
|--------|--------|----------------|
| Memory service provider | `healthCheck().components.memoryService.provider` | Unexpected `file` when `postgres` expected |
| Event bus provider | `healthCheck().components.eventBus.provider` | Unexpected `file` when `bullmq` expected |
| Fallback events | Console logs containing "falling back to file-based" | Any occurrence |
| PostgreSQL connection errors | Console logs `[Database]` | Any error |
| Redis connection errors | Console logs `[BullMQEventBus]` | Any error |
