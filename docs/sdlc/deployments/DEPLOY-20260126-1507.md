## 2. Installation Instructions

### 2.1 Global Installation (Recommended for Developers)

```bash
# Install globally
npm install -g @deltek/governance-engine

# Verify installation
governance --version

# Run validation
governance validate
```

### 2.2 Project Installation

```bash
# Install as dev dependency
npm install --save-dev @deltek/governance-engine

# Add to package.json scripts
{
  "scripts": {
    "governance:check": "governance check",
    "governance:validate": "governance validate --all",
    "governance:report": "governance report --format markdown"
  }
}
```

### 2.3 Git Hooks Setup (Husky)

```bash
# Install husky
npm install --save-dev husky

# Initialize husky
npx husky init

# Add pre-commit hook
echo 'npx governance check' > .husky/pre-commit
chmod +x .husky/pre-commit
```

## 3. Configuration

### 3.1 Policy File Location

The engine looks for policies in these locations (in order):

1. `~/.claude/governance/policies/org/deltek-engineering.yaml` (organization default)
2. `.governance/policy.yaml` (project override)
3. `governance-policy.yaml` (project root)

### 3.2 Create Default Policy Directory

```bash
mkdir -p ~/.claude/governance/policies/org
cp docs/sdlc/architecture/governance-policy-template.yaml \
   ~/.claude/governance/policies/org/deltek-engineering.yaml
```

### 3.3 Project-Specific Override

```yaml
# .governance/policy.yaml
version: "1.0.0"
name: "Project Override"
extends:
  - "~/.claude/governance/policies/org/deltek-engineering.yaml"

# Override specific settings
code_quality:
  test_coverage:
    minimum_total: 70  # Lower threshold for this project
```

## 4. Usage

### 4.1 CLI Commands

```bash
# Validate staged files (pre-commit)
governance check

# Validate specific files
governance validate -f src/myfile.ts

# Validate all files
governance validate --all

# Validate with specific policy
governance validate -p custom-policy.yaml

# Validate with bypass
governance validate --bypass --bypass-reason "Emergency fix approved by CTO"

# Generate report
governance report --format markdown -o COMPLIANCE.md

# JSON output for CI
governance validate --json
```

### 4.2 CI/CD Integration

#### GitHub Actions

```yaml
# .github/workflows/governance.yml
name: Governance Check

on: [push, pull_request]

jobs:
  governance:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx governance validate --all --json > governance-report.json
      - name: Upload Report
        uses: actions/upload-artifact@v4
        with:
          name: governance-report
          path: governance-report.json
      - name: Check Results
        run: |
          if [ $(jq '.passed' governance-report.json) != "true" ]; then
            echo "Governance check failed"
            exit 1
          fi
```

#### GitLab CI

```yaml
# .gitlab-ci.yml
governance:
  stage: test
  image: node:20
  script:
    - npm ci
    - npx governance validate --all
  artifacts:
    when: always
    reports:
      dotenv: governance.env
```

## 5. SDK Integration

### 5.1 Agent Integration

```typescript
import { GovernanceClient } from '@deltek/governance-engine';

const client = new GovernanceClient();

// Get policy injection for agent prompt
const policyContext = await client.getPromptInjection();
console.log(policyContext);

// Validate agent output before committing
const result = await client.validateOutput({
  agentType: 'software-engineer',
  files: [
    { path: 'src/domain/User.ts', content: '...' }
  ]
});

if (!result.passed) {
  console.error('Violations:', result.violations);
}
```

### 5.2 Programmatic Usage

```typescript
import { GovernanceService, validate } from '@deltek/governance-engine';

// Quick validation
const result = await validate({ staged: true });

// Full service usage
const service = new GovernanceService();
const result = await service.validate({
  files: ['src/index.ts'],
  policyPaths: ['custom-policy.yaml'],
  enforcement: {
    dryRun: true
  }
});
```
