## Deployment Architecture

```
Claude Code (User)
    |
    |--- hooks/user-prompt-submit.js  (UserPromptSubmit hook)
    |--- hooks/stop.js                (Stop hook)
    |--- hooks/post-write.js          (PostToolUse Write hook)
    |
    |--- MCP Server (stdio)
    |       |--- 8 tools (classify, start-workflow, etc.)
    |       |--- 3 resources (registry, health, config)
    |       |--- 4 prompts
    |
    |--- File IPC Bridge (~/.aisdlc/bridge/)
    |       |--- cache/      (classification cache)
    |       |--- messages/   (hook-MCP messages)
    |
    |--- Registry (~/.aisdlc/registry/)
            |--- workflows/  (workflow tracking)
            |--- requests/   (request log)
            |--- completions/(completion log)
            |--- file-changes/ (audit trail)
```


### Step 4: Configure Claude Code Hooks

Add to Claude Code settings (`~/.claude/settings.json` or project `.claude/settings.json`):

```json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "node /Users/gauravjetly/aisdlc-2.1.0/src/integration/dist/hooks/user-prompt-submit.js"
          }
        ],
        "timeout": 5000
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "node /Users/gauravjetly/aisdlc-2.1.0/src/integration/dist/hooks/stop.js"
          }
        ],
        "timeout": 3000
      }
    ],
    "PostToolUse": [
      {
        "matcher": {
          "tool_name": "Write"
        },
        "hooks": [
          {
            "type": "command",
            "command": "node /Users/gauravjetly/aisdlc-2.1.0/src/integration/dist/hooks/post-write.js"
          }
        ],
        "timeout": 5000
      }
    ]
  }
}
```

### Step 5: Configure MCP Server

Add to Claude Code MCP configuration:

```json
{
  "mcpServers": {
    "aisdlc-platform": {
      "command": "node",
      "args": ["/Users/gauravjetly/aisdlc-2.1.0/src/integration/mcp-server/dist/index.js"],
      "env": {
        "AISDLC_ROOT": "/Users/gauravjetly/aisdlc-2.1.0"
      }
    }
  }
}
```

### Step 6: Verify Installation

## Configuration

### Default Configuration

The integration ships with sensible defaults:

| Setting | Default | Description |
|---------|---------|-------------|
| `enabled` | `true` | Integration active |
| `autoClassify` | `true` | Auto-classify every message |
| `governance.level` | `2` | Light governance (advisory) |
| `classification.tier1Enabled` | `true` | Rule-based classification |
| `classification.tier2Enabled` | `true` | LLM classification fallback |
| `classification.confidenceThreshold` | `0.7` | Tier 2 trigger threshold |
| `performance.maxClassificationTime` | `3000` | Classification timeout (ms) |

### Custom Configuration

Create `~/.aisdlc/config.json`:

```json
{
  "enabled": true,
  "governance": { "level": 2 },
  "classification": {
    "tier1Enabled": true,
    "tier2Enabled": false,
    "confidenceThreshold": 0.7
  },
  "performance": { "maxClassificationTime": 3000 }
}
```

### Environment Overrides

| Variable | Effect |
|----------|--------|
| `AISDLC_ENABLED=false` | Kill switch: disables all integration |
| `AISDLC_GOVERNANCE_LEVEL=3` | Set governance level (1-4) |

## Rollback Plan

### Disable Integration (Quick)

Set environment variable:
```bash
export AISDLC_ENABLED=false
```

### Remove Hooks (Full Rollback)

Remove the `hooks` section from Claude Code settings.json. Claude Code will function normally without hooks.

### Remove MCP Server

Remove the `aisdlc-platform` entry from `mcpServers` in settings.json.
